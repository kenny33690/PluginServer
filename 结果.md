# 完成情况

已按照`设计流程.md`完成核心任务，具体如下：

1. 已将服务端接入 SignalR。
实现位置：`internal/app/app.go`、`internal/plugin/hub.go`

2. 已增加证书校验流程。
实现内容：
- 读取私有CA证书`root-ca.pem`（可通过`ROOT_CA_PATH`配置）
- 校验插件证书是否为CA签发
- 校验证书是否在有效期内
- 校验证书`Subject CommonName`是否与请求中的`name`一致

实现位置：`internal/plugin/cert.go`

3. 已采用 SQLite 存储插件注册信息。
实现内容：
- 自动初始化表`PluginRegistry`
- 保存字段包括`name`、`cert_string`、`subject`、`not_before`、`not_after`、`created_at`

实现位置：`internal/plugin/registry.go`

4. 已实现`createPlugin`请求处理。
方法名：`CreatePlugin(name, certString)`
处理流程：
- 校验参数非空
- 执行证书校验
- 校验通过后写入`PluginRegistry`
- 返回`plugin created`

实现位置：`internal/plugin/hub.go`

5. 已补充运行配置。
支持以下环境变量：
- `LISTEN_ADDR`，默认`:29100`
- `HUB_PATH`，默认`/ws`
- `ROOT_CA_PATH`，默认`root-ca.pem`
- `SQLITE_PATH`，默认`pluginserver.db`

实现位置：`internal/config/config.go`
